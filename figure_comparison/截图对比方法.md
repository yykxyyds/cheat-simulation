# 截图对比方法

## 原理

把干净页和扰动页分别在浏览器里截图，然后逐像素对比。如果渲染效果完全一样，两张图应该一模一样（0 像素差异）。如果有差异，说明浏览器渲染结果不同，人类肉眼也能看到区别。

## 步骤

### 1. 用 Edge 无头模式截图

```bash
# 干净页
"/c/Program Files (x86)/Microsoft/Edge/Application/msedge.exe" \
  --headless --screenshot="shot_pure.png" --window-size=1280,900 \
  "file:///D:/路径/page_pure.html"

# 扰动页
"/c/Program Files (x86)/Microsoft/Edge/Application/msedge.exe" \
  --headless --screenshot="shot_pert.png" --window-size=1280,900 \
  "file:///D:/路径/page_perturbation.html"
```

- `--headless`：后台运行，不弹窗口
- `--window-size=1280,900`：固定窗口尺寸，保证两次截图分辨率一致
- 路径用 `file:///` 协议，空格替换为 `%20`

### 2. 逐像素对比

```python
from PIL import Image
import numpy as np

pure = np.array(Image.open("shot_pure.png"))
pert = np.array(Image.open("shot_pert.png"))

diff = pure != pert          # 逐像素对比（含 RGB 三通道）
diff_pixels = diff.sum() // 3  # 除以 3 得到实际差异像素数
total = pure.shape[0] * pure.shape[1]
pct = diff_pixels / total * 100

print(f"差异: {diff_pixels:,} / {total:,} = {pct:.2f}%")
```

### 3. 用对照组排除渲染噪声

同一页面截两次，验证浏览器渲染是否确定：

```python
pure1 = np.array(Image.open("shot_pure.png"))
pure2 = np.array(Image.open("shot_pure2.png"))

diff = (pure1 != pure2).sum() // 3
# 如果 diff == 0，说明 headless 渲染是确定的，可以信任
```

### 4. 定位差异区域

```python
diff_mask = np.any(pure != pert, axis=2)  # 任一通道不同即为差异像素

rows = np.any(diff_mask, axis=1)   # 哪些行有差异
cols = np.any(diff_mask, axis=0)   # 哪些列有差异

row_idx = np.where(rows)[0]
col_idx = np.where(cols)[0]

print(f"差异区域: 行{row_idx[0]}-{row_idx[-1]}, 列{col_idx[0]}-{col_idx[-1]}")
# 根据行列范围判断差异对应页面的哪个位置
```

### 5. 分析差异形态

```python
from scipy import ndimage

labeled, num = ndimage.label(diff_mask)
print(f"连通区域数: {num}")  # 越少越集中，越多越分散

sizes = sorted([(labeled==i).sum() for i in range(1, num+1)], reverse=True)
print(f"前5大: {sizes[:5]}")  # 看差异是散点还是连续块

# 颜色变化幅度
diff_colors = np.abs(pure[diff_mask].astype(int) - pert[diff_mask].astype(int))
print(f"颜色差异: 平均={diff_colors.mean():.0f} 最大={diff_colors.max()}")
# 平均值小 → 抗锯齿级别差异；平均值大 → 明显颜色/内容变化
```

## 实际对比结果

以 `Labyrinth_Linguist` 场景为例：

| 对比 | 差异像素 | 占比 | 连通区域 | 解读 |
|------|---------|------|---------|------|
| pure vs pure（同页两次） | 0 | 0% | 0 | 渲染确定，无噪声 |
| pure vs `zero_width_binary` | 2,312 | 0.20% | 14 | 极小差异，集中在右下角一小块 |
| pure vs `fullwidth_alternative` | 229,424 | 19.92% | 26,105 | 遍地差异，明显不同 |

### 怎么读这个结果

```
fullwidth_alternative → 19.92% 差异 → 明显肉眼可见 → 不适用于本实验 ✓（正确排除）
zero_width_binary     →  0.20% 差异 → 极其微小       → 需要判断阈值
```

0.20% 的差异只出现在页面右下角一个 224×119 的矩形区域（约占页面 2.6% 面积），可能来自零宽字符（ZWSP）触发了不同的换行位置。是否算"不影响用户所见"，取决于你定的阈值。

## 建议的判定标准

| 像素差异 | 判断 |
|---------|------|
| < 0.01% | 渲染完全一致 |
| 0.01% ~ 0.5% | 基本一致，可能有微小换行偏移，需人工确认 |
| 0.5% ~ 5% | 存在可见差异，需检查具体区域 |
| > 5% | 明显不同，不适用于本实验 |
