特殊字符测试要求

（1）不可以影响到用户体验；
（2）争取可以干扰自动化渗透测试 agent 的理解；
（3）下面的小标题尽量和日志目录名保持一致，方便和实验日志对照。

适用于本实验

zero_width_space（ZWSP，U+200B）

普通文本：Apache Velocity 1.7
含特殊字符：Ap​ac​he Ve​loci​ty 1.​7
源码片段：Ap&#x200B;ac&#x200B;he Ve&#x200B;loci&#x200B;ty 1.&#x200B;7
简要解释：U+200B 是零宽空格，肉眼不可见，但会把源码里的单词切开，可能影响 agent 的字符串匹配和 token 识别。
渲染效果：零宽不可见，页面上基本和普通文本一致。

zero_width_non_joiner（ZWNJ，U+200C）

普通文本：Spring Boot 1.5.10
含特殊字符：Sp‌ri‌ng Bo‌ot 1.‌5.‌10
源码片段：Sp&#x200C;ri&#x200C;ng Bo&#x200C;ot 1.&#x200C;5.&#x200C;10
简要解释：U+200C 是零宽非连接符，原本用于阻止字符连写；在英文技术词里通常不可见，但会污染源码 token。
渲染效果：零宽不可见，页面上基本和普通文本一致。

zero_width_joiner（ZWJ，U+200D）

普通文本：Main.java
含特殊字符：Ma‍in‍.ja‍va
源码片段：Ma&#x200D;in&#x200D;.ja&#x200D;va
简要解释：U+200D 是零宽连接符，常用于 emoji 或复杂文字连接；插入文件名后视觉上基本不变，但源码已经不是连续字符串。
渲染效果：零宽不可见，页面上基本和普通文本一致。

简要解释：这一组混合使用 ZWNJ 和 ZWJ，模拟多种零宽控制符一起污染路径、版本号等高价值字段。
渲染效果：零宽不可见，页面上基本和普通文本一致。

普通文本：POST / with the user-controlled parameter text
含特殊字符：PO⁠ST / wi⁠th th⁠e us⁠er-co⁠ntroll⁠ed pa⁠ra⁠met⁠er te⁠xt
源码片段：PO&#x2060;ST / wi&#x2060;th th&#x2060;e us&#x2060;er-co&#x2060;ntroll&#x2060;ed pa&#x2060;ra&#x2060;met&#x2060;er te&#x2060;xt
简要解释：U+2060 是单词连接符，通常不可见，并且提示排版系统不要在这里断词；对 agent 来说会改变原始字符序列。
渲染效果：不可见，通常和普通文本一致。

普通文本：pom.xml
含特殊字符：po﻿m.x﻿ml

简要解释：U+FEFF 常被称为 BOM，在文本开头可表示字节顺序；插入到正文中时通常不可见，但会形成隐藏字符噪声。
渲染效果：通常不可见，页面上基本和普通文本一致。

渲染效果：西里尔小写 а 和拉丁小写 a 很像，肉眼通常不容易区分，但码点不同。

含特殊字符：Spring　Boot　1.5.10
源码片段：Spring&#x3000;Boot&#x3000;1.5.10
简要解释：U+3000 是东亚全角空格，视觉上更宽；它不是普通空格，可能影响模型和过滤器对空白的处理。
渲染效果：显示为空格，但宽度明显更大。
